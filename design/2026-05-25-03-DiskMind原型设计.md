## 0. 设计系统基线

> 综合 **ui-ux-pro-max** skill 推荐 + 已有可行性报告 / 技术设计 HTML 的视觉语言。

### 0.1 风格基调

| 维度 | 决策 | 理由 |
| --- | --- | --- |
| **整体风格** | OLED Dark Mode + Micro-interactions | 桌面工具长时使用 · 减少视觉疲劳 · 与系统暗色模式天然契合 |
| **辅助风格** | Bento Grid + Data-Dense Dashboard | 信息密度高,卡片化布局 |
| **避免** | Skeuomorphism · Brutalism · 卡通插画 · emoji 图标 | 不专业 / 不可信 |

### 0.2 配色系统(承接已有 HTML 视觉)

| 角色 | 值 | 用途 |

| Background-0 | `#07090f` | 最底层 |

### 0.3 字体

- **英文**:`Plus Jakarta Sans`(skill 推荐 · 现代 SaaS 工具感)
- **中文**:`PingFang SC` / `Microsoft YaHei`(平台回退)
- **等宽**:`SF Mono` / `Cascadia Code`(代码、文件路径)
- **字重**:400 / 500 / 600 / 700
- **基础字号**:14px · 行高 1.6

### 0.4 间距 & 圆角

- 8px 网格系统
- 圆角:小 6px · 中 10px · 大 16px
- 卡片内边距:20–28px

### 1.4 TopBar

```
┌─────────────────────────────────────────────────────────────────────────┐
│  [≡]  > 总览                       [🔍 搜索] [✦●AI ⌘L] [☉] [⚙]          │
└─────────────────────────────────────────────────────────────────────────┘
  ↑      ↑                              ↑       ↑           ↑    ↑
  │      面包屑                         全局搜索 AI 按钮      通知  快速设置
  SidebarTrigger                                 (含状态徽章)
```

- **高度**:56px · 固定 · 模糊毛玻璃背景
- **左**:`<SidebarTrigger>` + 面包屑路径
- **右**:
  - 全局搜索(`Cmd/Ctrl+K`)
  - **AI 按钮**(`Cmd/Ctrl+L`):**集状态指示 + 抽屉触发于一体**
  - 通知铃铛
  - 设置入口

#### AI 按钮状态徽章(承载实时状态)

| 视觉 | 状态 | Tooltip |
| --- | --- | --- |
| `[✦●AI]` 紫色发光 + 绿点 | 云端连接正常 | "DeepSeek-V3 · 今日 23 · ¥0.69" |
| `[✦◆AI]` 蓝色 + 蓝点 | 本地 Ollama 运行 | "Qwen2.5-3B · 本地 · 端口 11434" |
| `[✦○AI]` 灰色 | 未配置 / 已暂停 | "AI 未启用 · 点击配置" |
| `[✦◐AI]` 黄色脉动 | 调用中 / 重试中 | "正在分析 N 个文件…" |
| `[✦!AI]` 红色 | 失败 / 超额 | "连接失败 · 点击查看" |

- **左键**:打开 AI 抽屉(§2.4)
- **右键 / 长按**:弹出快速菜单
  - 切换后端(选择已配置的 Provider)
  - 暂停 AI(本次会话不调用)
  - 测试连接
  - 跳转到 Reports/AI 使用(§2.7.3)
  - 跳转到 Settings/AI 配置(§2.8)
- **Tooltip 详情**(hover 0.5s):当前后端 · 当前模型 · 今日次数 · 累计成本

> **设计理念**:Sidebar 不展示 AI 状态卡,而是把"实时状态"压缩到一个按钮的颜色 + 徽章里。
> 用户大多数时间只需知道"AI 是不是好的、是不是在工作",不需要常驻数字。
> 想看数字的时候,hover 看 Tooltip,想看趋势的时候,进 §2.7.3。

> **AI 抽屉是全局组件**,挂在 App 根 layout,任何页面下都可通过:
> - 顶栏 `✦●AI` 按钮(左键)
> - 快捷键 `Cmd/Ctrl+L`(仿 Cursor)
> - 扫描结果行内 `[问 AI ▸]`
> - Disk Map 块右键菜单 `Ask AI`
>
> 唤起,**永不离开当前页面**。

// stores/ai.ts (Pinia)
export const useAiStore = defineStore('ai', () => {
  const open = ref(false)
  const messages = ref<Message[]>([])
  const contextFiles = ref<FileEntry[]>([])

  function toggle() { open.value = !open.value }
  function addContext(f: FileEntry) { contextFiles.value.push(f) }

  return { open, messages, contextFiles, toggle, addContext }
})

// composables/useShortcuts.ts
useEventListener('keydown', (e) => {
  if ((e.metaKey || e.ctrlKey) && e.key === 'l') {

## 3. 跨页面交互流

### 3.1 首次启动流(Onboarding)

```
启动 → 引导页 1 (欢迎)
       → 引导页 2 (扫描权限申请 · Full Disk Access)
       → 引导页 3 (AI 模式选择)
       → 引导页 4 (隐私字段确认)
       → 跳转 Dashboard · 自动触发首次扫描
```

### 3.2 标准清理流

```
Dashboard
  ↓ [扫描磁盘]
Scan In Progress
  ↓ 完成 toast
Scan Results
  ↓ 用户勾选 · [清理已勾选]
AlertDialog 二次确认
  ↓ 确认
Cleaning (loading)
  ↓
Trash 完成态 (toast: "已释放 X GB,可在沙箱恢复")
  ↓
Dashboard (上次扫描卡片更新)
```

### 3.3 风险文件确认流

```
Scan Results
  ↓ 用户尝试勾选 ●高 风险文件
Tooltip 提示:"该文件位于用户文档目录,3 天前访问过"
  ↓ 强行勾选
AlertDialog:"你选择了 N 个高风险文件,逐项确认?"
  ↓ 确认
自动唤起 AI 抽屉(§2.4),把这 N 个文件注入"当前上下文"
  ↓ 用户在抽屉里逐项问 AI · 回复带 [应用建议] 按钮
  ↓ 主页面表格行随 AI 建议实时勾选 / 取消
```

### 3.4 AI 抽屉调用流(全局)

```
任意页面
  ↓ 任一触发(顶栏 ✦ AI / ⌘L / [问 AI ▸] / Disk Map 右键)
AI 抽屉从右侧滑入(主页面不变)
  ↓ 用户提问
流式回答 + [应用建议] 按钮
  ↓ 点击应用建议(可选)
主页面状态相应变化(如自动勾选文件)
  ↓ 关闭抽屉(Esc / × / ⌘L)
回到主页面 · 抽屉对话保留 · 下次打开继续
```

### 3.5 恢复流

```
Trash
  ↓ 用户勾选批次 · [恢复]
Toast loading
  ↓
Toast success: "已恢复 N 个文件到原路径"
  ↓
若原路径冲突 → Dialog 让用户选择(覆盖 / 重命名 / 跳过)
```

## 5. 关键状态机汇总

```
应用全局状态:
  Idle ──[扫描]──→ Scanning ──[完成]──→ Reviewing ──[清理]──→ Cleaning ──→ Done ──→ Idle
                       ↓                        ↓                ↓
                    [取消]                   [取消]           [失败]
                       ↓                        ↓                ↓
                      Idle                     Idle           Reviewing

AI 后端状态:
  Connected ──[网络失败]──→ Retrying ──[3 次失败]──→ Fallback (本地) ──[本地失败]──→ Rule-Only Mode
       ↑                                                                                  │
       └──────────────────[网络恢复]──────────────────────────────────────────────────────┘

Trash 状态:
  In_Trash ──[30 天 TTL]──→ Hard_Deleted
           ──[用户恢复]──→ Restored
           ──[文件占用]──→ Pending ──[重试成功]──→ Hard_Deleted
```
