## 一、项目概述

### 1.1 项目名称
**DiskMind**（磁盘智能体）—— 用大模型重新定义磁盘清理。

### 1.2 产品定位
一款面向 **Windows / macOS 双端**的桌面级智能磁盘分析与清理工具，通过本地扫描引擎采集文件元数据，结合大语言模型（LLM）对文件性质、可清理性、风险等级进行智能研判，提供"看得懂、敢放心删"的清理体验。

### 1.3 核心价值主张
| 维度 | 传统清理软件 | DiskMind |
| --- | --- | --- |
| 识别方式 | 规则库匹配（路径/扩展名） | 规则库 + LLM 语义判定 |
| 风险评估 | 二元（可删/不可删） | 多级风险评分 + 自然语言解释 |
| 用户认知 | 黑盒，用户被动信任 | 透明，每个建议附"为什么" |
| 长尾文件 | 难以识别非标准产物 | LLM 推断未知文件用途 |
| 个性化 | 千人一面 | 基于使用习惯的个性化建议 |

### 1.4 一句话商业故事
> "当 CCleaner 还在比谁的规则库更大时，DiskMind 让每台电脑都有一个懂你磁盘的 AI 管家。"

### 5.3 大模型接入策略（核心）

**双轨制：用户可选「云端高质量」或「本地隐私优先」。**

| 模式 | 模型 | 优点 | 缺点 | 适用人群 |
| --- | --- | --- | --- | --- |
| 云端模式 | DeepSeek-V3 / GPT-4o-mini / Claude Haiku | 准确率高、上下文长、便宜 | 需网络、有隐私顾虑 | 大多数用户 |
| 本地模式 | Qwen2.5-3B / Phi-3.5 / Llama 3.2 3B (通过 Ollama) | 全本地、零外发 | 占内存、速度慢 | 隐私敏感、企业 |
| 混合模式 | 文件元数据本地推理，复杂决策走云端 | 平衡 | 工程复杂 | 推荐默认 |

**关键工程做法：**
1. **不发送文件内容**，仅发送：路径 / 大小 / 修改时间 / 扩展名 / 应用归属 / 是否在系统目录等元数据。
2. **批量打包 prompt**：一次请求评估多个文件，降低 token 消耗。
3. **强 JSON Schema 输出**：返回 `{path, risk, reason, action}`，结构化解析。
4. **结果缓存**：相同文件指纹 + 模型版本 → 复用结论，避免重复消费。
5. **降级方案**：网络异常时降回纯规则模式。

### 5.4 扫描引擎关键设计

- **多线程深度遍历**：基于 Rust `rayon`，按磁盘 IO 队列调度，避免 SSD 与 HDD 反应差异。
- **跳过策略**：自动跳过 `node_modules` 内的二级 nesting、系统保护目录、用户白名单。
- **哈希算法**：先 4KB 头部 SHA-1 快速去重，再全量 BLAKE3 精校。
- **元数据索引**：增量写入 SQLite，支持秒级二次查询。
- **后台 watcher**（可选）：基于 `notify` crate 监听变更，实现"零等待"再扫描。

### 5.5 安全与隐私

- **删除即沙箱**：所有"删除"先入应用内 trash（30 天），再异步真删。
- **白名单保护**：系统关键目录、近 7 天活跃文件、用户 pin 文件，**永不删除**。
- **可回滚日志**：每次操作生成 manifest，支持 `undo`。
- **隐私模式**：完全本地、零网络请求。
- **数据上传白名单审计**：可视化展示发往云端 LLM 的全部字段。

## 六、AI 大模型应用价值（差异化核心）

### 6.1 LLM 在本产品中能做的四件事

#### ① 未知文件性质判定
传统规则库无法识别 `~/.config/random-tool-xyz/cache.db` 这类长尾文件。LLM 可基于路径、上下文、相邻文件，推断其归属与可清理性。

#### ② 风险评分 + 自然语言解释
```json
{
  "path": "/Users/x/Library/Caches/com.adobe.AdobePhotoshop/PSCache",
  "size_mb": 4320,
  "risk_score": 12,
  "risk_level": "low",
  "category": "应用缓存",
  "explanation": "Adobe Photoshop 自动重建的缓存目录，删除后下次启动会重新生成，仅影响首次加载速度。建议清理。",
  "action": "safe_to_delete"
}
```

#### ③ 自然语言问答
用户问："这个 6GB 的 `Xcode/DerivedData` 能删吗？"
LLM 答："可以。Xcode 会在下次编译时自动重建，删除不会影响项目源码，仅会让下次构建慢一些。"

#### ④ 个性化策略
通过用户清理行为（保留/删除/恢复）持续微调建议权重，做到"越用越懂你"。

### 6.2 Prompt 工程范式

**System Prompt 核心约束：**
- 永远输出 JSON
- 不臆造文件信息，证据不足时返回 `unknown` + 解释
- 风险评分必须给出 0–100 的数值与依据
- 中文输出，避免技术黑话

**Few-shot 示例驱动**：内置 50+ 高频文件类型样例，校准模型一致性。

### 6.3 成本测算

- 单文件元数据 ≈ 80 tokens
- 单次扫描典型 5 万个待评估文件
- 批量 50 文件/请求 → 1000 次请求
- DeepSeek-V3：约 **¥0.03 / 次完整扫描**
- 月活 10 万用户、人均 4 次/月扫描 → **月成本 ≈ ¥1.2 万**，完全可承受

## 十、结论与建议

### 10.1 结论
**强烈建议立项**。理由如下：
1. **市场空间存在**且竞品老化，AI 化是未灭团队都未真正吃下的红利。
2. **技术可行性已具备**：Tauri + Rust + LLM API 的组合在 2026 年完全成熟。
3. **差异化清晰**：不是"再做一个 CCleaner"，而是"让磁盘清理这件事被 AI 重写"。
4. **商业模型已被验证**：CleanMyMac 同等付费模式已成立，DiskMind 拥有跨平台与 AI 双重溢价空间。

### 10.2 关键成功要素（CSF）
1. **零误删神话**：上线第一年不能有任何一例核心文件误删事故。
2. **AI 解释要让人懂**：自然语言解释必须做到普通用户读得懂、信得过。
3. **品牌叙事**：把"AI 管家"形象立住，对抗大厂内置功能的同质化竞争。
4. **跨平台体验**：Mac 用户与 Win 用户得到一致的视觉与心智锚点。
