#### S1 详解:i18n 审计

- **范围**:`grep` 全量审计 `app/src/{pages,components,layouts}/**/*.vue` 中的硬编码中文字符串(中文 Unicode 范围 `[\u4e00-\u9fa5]`),排除 i18n 字典本身、注释、`v-text`
- **预期命中**:placeholder / aria-label / tooltip / error message / button text 等长尾位
- **执行步骤**:
  1. `rg "[\u4e00-\u9fa5]" app/src/{pages,components,layouts} --type vue --line-number -A 0 | grep -v "/\\*\\|//"` 输出候选清单
  2. 按文件分组,逐个判定哪些应该走 i18n
  3. 在 `i18n/zh-CN.ts` + `en-US.ts` 补 key
  4. 替换组件中的硬编码为 `t('key')`
  5. `vue-tsc --noEmit` 校验
- **验收**:`rg` 输出从 X 行降至 < 10 行(剩余的应为合理硬编码,如 mock data 等)

#### S2 详解:`MAX_SCAN_HISTORY` 设置项化

- **当前**:`db/mod.rs` 中 `const MAX_SCAN_HISTORY: i64 = 30;` 硬编码
- **目标**:迁到 `meta` 表(参考 `trash_retention_days` 的实现),设置 → 通用 加 number input(范围 10-200,默认 30)
- **执行步骤**:
  1. 后端:`db::max_scan_history()` / `set_max_scan_history()` 函数 + IPC `meta_get_max_scan_history` / `meta_set_max_scan_history`
  2. `save_scan` 事务末尾的 `DELETE FROM scan_run WHERE id NOT IN (... LIMIT MAX_SCAN_HISTORY)` 改用动态值
  3. 前端 `GeneralSettingsTab` 加 NumberInput + 即时保存 + toast 反馈
  4. i18n 加 key `settings.general.maxScanHistory*`
- **验收**:用户修改后下次扫描立即生效(无需重启),DB 立即裁剪

| **S7** | 前端 `onErrorCaptured` 收口 | 2.5 | ~2 小时 | 极低 | ⭐⭐⭐ | `lib/notify.ts` 现有 toast(已就位) |

#### S5 详解:Cmd+K Command Palette

- **目标**:类似 VSCode / Linear 的全局命令面板,Cmd+K 弹起,支持模糊搜索:
  - 页面跳转(仪表盘 / 扫描 / 报告 / 回收站 / 设置)
  - 设置项跳转(隐私 / AI / 主题 / 语言)
  - 最近扫描结果中的文件
  - 触发动作(开始扫描 / 终止扫描 / 打开 AI 助手 / 切换语言 / 切换主题)
- **依赖**:
  - reka-ui `Combobox` 或 `Command` 组件(需调研)
  - 索引数据源汇总:`stores/scan.ts` results + 路由表 + 设置 anchor 链接
- **执行步骤**:
  1. 调研 reka-ui Combobox API(可能要额外引入 `cmdk-vue` 之类)
  2. 全局快捷键注册(`composables/useHotkeys.ts` 或 VueUse `useMagicKeys`)
  3. 命令源聚合:`composables/useCommandPalette.ts` 返回 `{ id, label, icon, action }[]`
  4. 弹层组件 `components/layout/CommandPalette.vue` + Dialog + 输入框 + 列表
  5. i18n 全部 key
- **风险**:索引数据源类型多样,排序/去重/分组逻辑可能复杂
- **可分阶段**:先做"页面跳转"基础版,再迭代加"扫描结果搜索"

#### S6 详解:Rust panic 捕获

- **目标**:`std::panic::set_hook` 把 panic 写入 `app_data_dir/crash.log`,带时间戳 + backtrace + 进程上下文
- **执行步骤**:
  1. `lib.rs::run` 顶部调用 `std::panic::set_hook(Box::new(panic_handler))`
  2. `panic_handler` 拼接 panic info + `std::backtrace::Backtrace::capture()` + 写入 `app_data_dir/crash.log`(append 模式 + 日期分割)
  3. 提供 IPC `read_crash_log` 让前端能展示
  4. 设置 → 通用 加"打开崩溃日志目录"按钮(复用 `reveal_in_explorer`)
- **风险**:`Backtrace::capture()` 在 release build 默认禁用,需要 `RUST_BACKTRACE=1` 或 `Cargo.toml` 配置 `debug = true`
- **进阶**:用户同意后,加 Sentry SDK / 自建上报 endpoint(留作 S13 单独决策)

#### S7 详解:前端 onErrorCaptured

- **目标**:`App.vue` 顶层 `onErrorCaptured((err, instance, info) => { notify.error(...); logToFile(err); return false })` 拦截所有 Vue 组件树异常
- **执行步骤**:
  1. `App.vue` 加 `onErrorCaptured` 钩子
  2. 走 `notify.error()`(已实现)显示 toast
  3. 可选:也写入 `app_data_dir/frontend-error.log`(借助 `tauri` IPC)
- **与 S6 合并**:统一一个"崩溃 / 异常落本地"接口,前后端共用

**Sprint 2 总工作量**:~3-4 天 · **目标:M5 进度 70%(Header 增强 + 错误监控基本闭环)**

| **S15** | 备份/导出 `scan_run` → JSON/CSV | 1.5 | ~半天 | 与 S9 实现思路同 | ⭐⭐⭐ 做 |

#### S14 详解:重复文件检测(BLAKE3 两阶段)

- **目标**:解锁 `ScanOptionsCard.vue` 的 `detectDuplicates` 开关(当前是装饰)
- **算法**:两阶段哈希降低 IO 成本
  1. **第一阶段**:相同 size 的文件分组(scan 时已记录 `size_bytes`)
  2. **第二阶段(可选)**:对 size 相同的候选,读前 64KB(BLAKE3 head hash)再分组
  3. **第三阶段**:对剩余仍冲突的,读全文件 BLAKE3 确认
- **执行步骤**:
  1. 后端:`scanner/dedup.rs` 新增模块,接 `Vec<ScanResultRow>` 返回 `Vec<DuplicateGroup>`
  2. `ScanResultRow` 加可选 `hash_head` / `hash_full` 字段
  3. DB:`scan_result` 加两列(或新建 `dup_group` 表)
  4. 前端:Scan 页 Tab 加"重复文件"视图(reka-ui Tree 按 group 展开)
  5. 开关接真:`computeHash` + `detectDuplicates` 之一开启时,scan 完毕后跑 dedup pipeline
- **风险**:大量大文件场景下 IO 成本飙升,需限速 + 进度反馈
- **依赖库**:`blake3` crate(已是事实标准,~5KB binary 增量)

#### S15 详解:备份/导出 `scan_run`

- **目标**:`reports` 页或设置页加"导出扫描历史"按钮,选择 JSON / CSV 格式
- **执行步骤**:
  1. 后端:`export_scan_runs(format: "json" | "csv") -> String` IPC,返回完整 dump
  2. 前端:tauri `save` dialog 选保存路径 + toast 反馈
  3. CSV 字段:`run_id, started_at, finished_at, duration_ms, file_count, reclaimable_bytes, fingerprint`
- **可与 S9 合并**:同样思路实现 AI 日志导出 + 扫描历史导出,共享底层 CSV 生成 helper

**Sprint 3 总工作量**:~6-9 天(看是否完整做 S14)· **目标:M5 完整收官 + M6 进度 30-50%**
